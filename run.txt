================================================================================
  1. PROJECT OVERVIEW
================================================================================

The Food Calorie Estimator is a deep learning-powered web application that
allows users to upload a photograph of food and receive:
  - The name of the food item (from 5 supported classes)
  - The estimated calorie count per serving
  - The model's confidence percentage in its prediction

This project demonstrates the full ML pipeline: from data preparation and model
training, to serving predictions through a web API and a browser-based UI.

================================================================================
  2. SUPPORTED FOOD CLASSES
================================================================================

The model is trained to recognize the following 5 food categories:

  1. 🍔 Hamburger       — ~295 calories per serving
  2. 🍕 Pizza           — ~285 calories per serving
  3. 🍟 French Fries    — ~312 calories per serving
  4. 🥪 Club Sandwich   — ~250 calories per serving
  5. 🥗 Caesar Salad    — ~150 calories per serving

These classes are from the Food-101 benchmark dataset. The calorie values are
average per-serving estimates stored in the calories.json database.

================================================================================
  3. MACHINE LEARNING — THEORY & CONCEPTS USED
================================================================================

--- 3.1 Transfer Learning ---
  Transfer Learning is an ML technique where a model trained on one large
  dataset is reused as the starting point for a model on a different task.
  Instead of training from scratch (which requires millions of images and
  weeks of compute), we take a model pre-trained on ImageNet (1.2 million
  images, 1000 classes) and adapt it to our 5-class food problem.

  WHY: Our dataset is small (~200 images per class). Training from scratch
  would lead to overfitting. Transfer learning lets us leverage learned
  low-level features (edges, textures, shapes) from ImageNet.

--- 3.2 MobileNetV2 (Base Architecture) ---
  MobileNetV2 is a lightweight Convolutional Neural Network (CNN) architecture
  developed by Google. It uses:
    - Inverted Residual Blocks: Expands channels, applies depthwise convolution,
      then projects back — very efficient on CPU/mobile hardware.
    - Depthwise Separable Convolutions: Splits standard convolution into two
      cheaper operations, drastically reducing parameters.
    - Linear Bottlenecks: Avoids non-linear activation in bottleneck layers to
      preserve information.

  Input shape  : 224 × 224 × 3 (RGB image)
  Pre-trained  : ImageNet weights (imagenet)
  Trainable    : FALSE (all base layers are FROZEN)

--- 3.3 Fine-Tuning / Feature Extraction ---
  We use the feature extraction mode of transfer learning:
    - The MobileNetV2 base is frozen (weights not updated during training)
    - Only our custom classification head is trained
  This prevents catastrophic forgetting and is faster to train.

--- 3.4 Custom Classification Head ---
  On top of the frozen MobileNetV2 base, we add:
    - GlobalAveragePooling2D : Converts the 7×7 feature map to a 1D vector
                               (reduces parameters, avoids overfitting)
    - Dense(128, ReLU)       : Fully connected layer, 128 neurons
    - Dense(5, Softmax)      : Output layer — 5 neurons for 5 food classes
                               Softmax gives probability distribution (sums to 1)

--- 3.5 CNN (Convolutional Neural Network) ---
  CNNs are the backbone of image classification. They learn hierarchical
  spatial features:
    - Early layers  : detect edges, corners, textures
    - Middle layers : detect object parts (circles, curves)
    - Deep layers   : detect high-level concepts (eyes, wheels, buns)
  MobileNetV2 is a specific CNN architecture optimized for speed.

--- 3.6 Data Augmentation ---
  To artificially expand training data and improve generalization, we apply
  random transformations to input images during training:
    - rotation_range=15     : Rotate image up to ±15 degrees
    - width_shift_range=0.1 : Horizontal shift up to 10%
    - height_shift_range=0.1: Vertical shift up to 10%
    - horizontal_flip=True  : Random left-right mirror
    - zoom_range=0.1        : Random zoom in/out by 10%
    - rescale=1/255         : Normalize pixel values from [0,255] to [0,1]

--- 3.7 Loss Function & Optimizer ---
  Loss     : Categorical Crossentropy
             Used for multi-class classification. Measures how far the predicted
             probability distribution is from the true one-hot label.

  Optimizer: Adam (Adaptive Moment Estimation)
             Combines momentum and RMSProp. Adapts learning rate per parameter.
             Learning Rate = 0.001

  Metric   : Accuracy (proportion of correctly classified images)

--- 3.8 Softmax & Argmax (Inference) ---
  At prediction time:
    - The model outputs a vector of 5 probabilities (via Softmax)
    - np.argmax() finds the class index with highest probability
    - That index maps to the food class name (CLASSES list, alphabetically sorted)
    - The max probability × 100 = confidence percentage shown to user

--- 3.9 Overfitting & Why We Used 200 Images/Class ---
  Overfitting occurs when a model memorizes training data and fails on new data.
  With only 200 images per class (1000 total), combined with:
    - Frozen base model (no full retraining)
    - Data augmentation
    - Dropout-like effect of GlobalAveragePooling
  ...we mitigate overfitting effectively for a demo-scale project.

--- 3.10 Training / Validation Split ---
  We use an 80/20 split:
    - 80% of images → Training set (model learns from these)
    - 20% of images → Validation set (model evaluated on these, unseen)
  This allows us to monitor val_accuracy to detect overfitting.

  5. PROJECT FILES — PURPOSE OF EACH FILE
================================================================================

  FILE                  PURPOSE
  ─────────────────────────────────────────────────────────────────────────────

  train_model.py        ML Training Script
                        - Reads Food-101 dataset images from dataset/images/
                        - Filters and copies 200 images per class into
                          dataset/filtered/
                        - Creates ImageDataGenerators for train and validation
                        - Loads MobileNetV2 (ImageNet weights, frozen)
                        - Adds custom Dense classification head
                        - Trains for 5 epochs using Adam + Categorical CE loss
                        - Saves trained weights to model/food_model.h5
                        RUN THIS ONCE before starting the web app.

  app.py                Flask Web Application (Backend)
                        - Loads the saved model (food_model.h5) at startup
                        - Loads calorie database (calories.json)
                        - Serves the web UI at http://localhost:5000

                        - Runs model.predict() and maps result to class name
                        - Returns food name, calorie count, confidence % to UI
                        - Saves uploaded image to static/uploads/ for display

  calories.json         Calorie Database
                        - Simple JSON key-value store
                        - Maps food class name → calories per serving (integer)
                        - Example: {"hamburger": 295, "pizza": 285, ...}
                        - Loaded once at app startup, queried per prediction

  templates/index.html  Frontend UI (Jinja2 Template)
                        - Full browser-based user interface
                        - Drag-and-drop or click-to-upload image area
                        - Food class badge pills (Hamburger, Pizza, etc.)
                        - Predict button to submit the form
                        - Displays result: food name, calories, confidence bar
                        - Displays uploaded image preview
                        - Error messages for invalid uploads or model issues
                        - Styled with dark theme, gradients, animations

  static/               Static Assets Folder
                        - static/uploads/ : Stores uploaded images temporarily
                          so the app can display the image alongside results
                        - Other CSS/JS assets if any

  dataset/              Training Dataset (NOT included in Docker image)
                        - dataset/images/ : Original Food-101 images per class
                        - dataset/filtered/ : Filtered 200 images/class used
                          for training (created by train_model.py)
                        - This folder is in .dockerignore — not needed to run

  model/                Trained Model Storage
                        - model/food_model.h5 : Saved Keras model in HDF5 format
                        - Contains all layer configs + trained weights
                        - Generated by train_model.py, loaded by app.py
                        - Included in Docker image

  Dockerfile            Docker Build Instructions
                        - Defines the container environment step by step
                        - Based on python:3.10-slim (lightweight Linux)
                        - Installs system libraries for TensorFlow + Pillow
                        - Installs Python packages from requirements.txt
                        - Copies app files into container
                        - Exposes port 5000
                        - Starts app with Gunicorn on container boot

  requirements.txt      Python Dependencies List
                        - Lists all pip packages and versions needed
                        - Used by Docker during build and for local setup
                        - Ensures reproducible environments

  .dockerignore         Docker Exclusion Rules
                        - Tells Docker what NOT to copy into the image
                        - Excludes: dataset/ (huge, not needed for inference)
                        - Excludes: __pycache__/, .git/, venv/, etc.

================================================================================
  6. HOW THE PROJECT WORKS — END TO END FLOW
================================================================================

  PHASE 1: TRAINING (run once locally)
  ─────────────────────────────────────
  1. train_model.py reads 200 images per class from Food-101 dataset
  2. ImageDataGenerator applies augmentation + normalization
  3. MobileNetV2 (pretrained, frozen) extracts image features
  4. Custom Dense head learns to classify those features into 5 classes
  5. Model trains for 5 epochs, saving best weights to food_model.h5

  PHASE 2: INFERENCE (every time user uploads an image)
  ──────────────────────────────────────────────────────
  1. User opens browser → http://localhost:5000
  2. User uploads or drag-drops a food image
  3. Flask receives the file via POST request
  4. app.py saves the file, opens it with Pillow
  5. Image is resized to 224×224 pixels (MobileNetV2 input size)
  6. Pixels normalized: divided by 255.0 → values in [0.0, 1.0]
  7. Batch dimension added: shape becomes (1, 224, 224, 3)
  8. model.predict() runs forward pass through the neural network
  9. Output: softmax vector of 5 probabilities, e.g. [0.02, 0.91, 0.04, 0.02, 0.01]
  10. np.argmax() → index 1 → 'club_sandwich' (alphabetical mapping)
  11. Confidence = max probability × 100 = 91%
  12. calories_db['club_sandwich'] = 250
  13. Flask renders index.html with result dict → shown to user

================================================================================
  9. KEY ML TERMS GLOSSARY
================================================================================

  TERM                   MEANING
  ─────────────────────────────────────────────────────────────────────────────
  CNN                    Convolutional Neural Network — detects spatial patterns
  Transfer Learning      Reuse weights from a model trained on a larger dataset
  Fine-Tuning            Unfreeze + retrain some layers of a pre-trained model
  Feature Extraction     Use frozen pre-trained model as a fixed feature extractor
  MobileNetV2            Lightweight CNN architecture by Google (efficient on CPU)
  ImageNet               Large dataset of 1.2M images, 1000 classes (used to
                         pre-train MobileNetV2)
  Epoch                  One full pass through all training data
  Batch Size             Number of images processed before weights update (=16)
  Softmax                Activation that converts raw scores to probabilities (sum=1)
  Argmax                 Returns index of the highest value in an array
  GlobalAveragePooling   Reduces feature map to vector by averaging spatial dims
  Overfitting            Model memorizes training data, fails on new data
  Augmentation           Random image transforms to artificially increase data
  Normalization          Scale pixel values [0,255] → [0,1] for stable training
  One-Hot Encoding       Class label as vector: class 2 of 5 → [0,0,1,0,0]
  Categorical CE Loss    Loss function for multi-class softmax classification
  Adam Optimizer         Adaptive learning rate optimizer (momentum + RMSProp)
  Inference              Using a trained model to make predictions on new data
  .h5 / HDF5 format      File format for saving entire Keras model (weights+config)
  WSGI                   Web Server Gateway Interface — protocol Gunicorn uses
  Gunicorn               Production Python WSGI HTTP server (replaces Flask dev)
  Docker                 Containerization tool — packages app + environment together
  Container              Isolated runtime environment created from a Docker image
